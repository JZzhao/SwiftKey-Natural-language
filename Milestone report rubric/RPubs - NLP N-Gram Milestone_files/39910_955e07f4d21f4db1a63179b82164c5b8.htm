Now we combine the three vectors into one and perform sentence detection to allow or n-gram models to predict across multiple lines.

sample_text <- paste(blogs, news, tweets)
sample_text <- sent_detect(sample_text, language = "en", model = NULL)

Creating the Corpus and cleaning the data

Now we create the corpus and remove out information that cannot be used for text prediction such as whitespace, numbers, punctuation, etc.

sample_corpus <- VCorpus(VectorSource(sample_text))
sample_corpus <- tm_map(sample_corpus, removePunctuation)
sample_corpus <- tm_map(sample_corpus, stripWhitespace)
sample_corpus <- tm_map(sample_corpus, removeNumbers)
sample_corpus <- tm_map(sample_corpus, content_transformer(tolower))

Remove profanity (list taken from online source)

profanity <- VectorSource(readLines(ProfanityLocation))
sample_corpus <- tm_map(sample_corpus, removeWords, profanity)

Convert the sample corpus to a data frame to plug into RWeka Tokenizer

sample_df <- data.frame(text=unlist(sapply(sample_corpus, '[',"content")),stringsAsFactors=F)

Create tokenizers

token_delim <- " \\t\\r\\n.!?,;\"()"

onetoken <- NGramTokenizer(sample_df, Weka_control(min=1,max=1))
bitoken <- NGramTokenizer(sample_df, Weka_control(min=2,max=2, delimiters = token_delim))
tritoken <- NGramTokenizer(sample_df, Weka_control(min=3,max=3, delimiters = token_delim))
quatrtoken <- NGramTokenizer(sample_df, Weka_control(min=4,max=4, delimiters = token_delim))

Now we setup the comparison data from our tokenizers to understand the data better

one_word <- data.frame(table(onetoken))
two_word <- data.frame(table(bitoken))
three_word <- data.frame(table(tritoken))

Analyzing Word Frequencies

Assuming all words used in the English language come from our sample of corpora (a very big assumption!), we can see in the following calculation how many words it would take to reach 50% and 90% coverage in our one word model.

sort_one_freq <- cbind(sort_one,sort_one$Freq/sum(sort_one$Freq))
colnames(sort_one_freq) <- c("Word", "Freq", "Pct_Freq")
running_freq = 0
words_counted = 0
while(running_freq<0.50){
  running_freq <- running_freq + sort_one_freq[(words_counted+1),3]
  words_counted <- words_counted + 1
  }

139 words are needed to account for 50% of the entire frequency corpora. This is the same as 0.658% of the rows in our one-word sample corpus.
